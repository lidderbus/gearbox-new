## 2. 齿轮箱选型逻辑

### 2.1 核心计算公式

#### 2.1.1 必要传递能力计算
```javascript
// 必要传递能力 (kW/rpm)
requiredTransferCapacity = enginePower / engineSpeed
```

#### 2.1.2 发动机扭矩计算
```javascript
// 发动机扭矩 (N·m)
engineTorque_Nm = (enginePower * 9550) / engineSpeed
```

#### 2.1.3 容量余量计算
```javascript
// 容量余量 (%)
capacityMargin = ((actualCapacity - requiredCapacity) / requiredCapacity) * 100
```

#### 2.1.4 安全系数计算
```javascript
// 安全系数
safetyFactor = actualCapacity / requiredCapacity
```

### 2.2 多层过滤条件

选型过程采用逐层过滤机制，顺序如下：

```
┌──────────────────────────────────────────────────────────────┐
│ 第1层：转速范围检查                                           │
│ ────────────────────                                         │
│ 条件：engineSpeed ∈ [minSpeed, maxSpeed]                     │
│ 说明：发动机转速必须在齿轮箱的输入转速范围内                      │
└──────────────────────────────────────────────────────────────┘
                              │ 通过
                              ▼
┌──────────────────────────────────────────────────────────────┐
│ 第2层：减速比偏差检查                                         │
│ ────────────────────                                         │
│ 条件：|ratio - targetRatio| / targetRatio × 100 ≤ 25%        │
│ 参数：MAX_RATIO_DIFF_PERCENT = 25                            │
│ 说明：选择与目标减速比偏差在25%以内的齿轮箱比例                   │
└──────────────────────────────────────────────────────────────┘
                              │ 通过
                              ▼
┌──────────────────────────────────────────────────────────────┐
│ 第3层：传递能力检查                                           │
│ ────────────────────                                         │
│ 条件1：capacity ≥ requiredTransferCapacity (必须满足需求)     │
│ 条件2：capacityMargin ≤ 50% (不能过度选型)                    │
│ 参数：MAX_CAPACITY_MARGIN = 50                               │
└──────────────────────────────────────────────────────────────┘
                              │ 通过
                              ▼
┌──────────────────────────────────────────────────────────────┐
│ 第4层：推力要求检查 (可选)                                     │
│ ────────────────────                                         │
│ 条件：gearbox.thrust ≥ thrustRequirement                     │
│ 说明：仅当用户指定推力要求时进行检查                            │
└──────────────────────────────────────────────────────────────┘
```

### 2.3 选型参数阈值

| 参数名称 | 阈值 | 说明 |
|---------|------|------|
| MAX_RATIO_DIFF_PERCENT | 25% | 减速比最大偏差百分比 |
| MAX_CAPACITY_MARGIN | 50% | 容量余量上限 |
| 理想容量余量 | 5-20% | 最佳容量余量范围 |
| 近似匹配减速比偏差 | ≤35% | 近似匹配的减速比阈值 |
| 近似匹配容量系数 | ≥0.85 | 容量至少达到需求的85% |
| 近似推力系数 | ≥0.8 | 推力至少达到需求的80% |

### 2.4 近似匹配机制

当无法找到完全符合条件的齿轮箱时，系统会收集"近似匹配"结果：

```javascript
// 近似匹配条件
近似匹配情况 = {
  减速比超范围: ratioDiffPercent ≤ 35%,      // 比正式条件宽10%
  容量略不足: capacity ≥ required × 0.85,   // 至少满足85%需求
  容量余量过高: margin ≤ MAX_MARGIN × 1.3,  // 最多超出30%
  推力略不足: thrust ≥ required × 0.8       // 至少满足80%推力
}
```

## 3. 联轴器选型逻辑

### 3.1 扭矩计算公式

#### 3.1.1 联轴器所需扭矩计算
```javascript
// 所需联轴器扭矩 (kN·m)
requiredCouplingTorque_kNm = (engineTorque_Nm × K × St) / 1000

// 其中：
// engineTorque_Nm = 发动机扭矩 (N·m)
// K = 工作条件系数
// St = 温度系数
```

### 3.2 工作条件系数 (K)

| 工作条件类别 | 系数值 | 描述 |
|------------|-------|------|
| I类:扭矩变化很小 | 1.0 | 平稳运行，负载均匀 |
| II类:扭矩变化小 | 1.2 | 轻微波动 |
| III类:扭矩变化中等 | 1.4 | 中等波动 (默认) |
| IV类:扭矩变化大 | 1.6 | 明显冲击 |
| V类:扭矩变化很大 | 1.8 | 严重冲击 |

### 3.3 温度系数 (St)

| 温度范围 | 系数值 |
|---------|-------|
| ≤20°C | 1.0 |
| 21-40°C | 1.1 |
| 41-60°C | 1.2 |
| 61-80°C | 1.3 |
| >80°C | 1.4 |

### 3.4 联轴器筛选条件

```
┌──────────────────────────────────────────────────────────────┐
│ 条件1：扭矩满足                                               │
│ ────────────────                                             │
│ coupling.torque ≥ requiredCouplingTorque_kNm                │
└──────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌──────────────────────────────────────────────────────────────┐
│ 条件2：转速满足                                               │
│ ────────────────                                             │
│ coupling.maxSpeed ≥ engineSpeed                              │
└──────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌──────────────────────────────────────────────────────────────┐
│ 条件3：罩壳要求 (可选)                                        │
│ ────────────────                                             │
│ 如果需要罩壳，筛选带'JB'、'J'或'-ZB'后缀的型号                  │
└──────────────────────────────────────────────────────────────┘
```

## 4. 备用泵选型逻辑

### 4.1 是否需要备用泵判断

```javascript
needsStandbyPump(gearboxModel, options) {
  // GW系列和GC系列：始终需要
  if (model.startsWith('GW') || model.startsWith('GC')) return true;

  // HC系列：仅特定型号需要
  if (model.startsWith('HC')) {
    seriesNum = parseInt(model.match(/HC(\d+)/)[1]);
    return (seriesNum >= 1000 && seriesNum <= 1200) ||
           (seriesNum >= 1400 && seriesNum <= 2000) ||
           (seriesNum >= 2700 && seriesNum < 2800);
  }

  // DT系列：特定范围需要
  if (model.startsWith('DT')) {
    seriesNum = parseInt(model.match(/DT(\d+)/)[1]);
    return (seriesNum >= 180 && seriesNum <= 770) ||
           (seriesNum >= 900 && seriesNum <= 1400) ||
           (seriesNum >= 1500 && seriesNum <= 2400);
  }

  // HCM/HCQ系列：≥300的型号需要
  if (model.startsWith('HCM') || model.startsWith('HCQ')) {
    seriesNum = parseInt(model.match(/(HCM|HCQ)(\d+)/)[2]);
    return seriesNum >= 300;
  }

  // 其他情况：功率≥600kW需要
  return options.power >= 600;
}
```

## 5. 价格计算逻辑

### 5.1 价格体系结构

```
┌─────────────────┐
│    基础价格      │  basePrice
│   (厂家定价)     │
└────────┬────────┘
         │ × (1 - 折扣率)
         ▼
┌─────────────────┐
│    工厂价格      │  factoryPrice = basePrice × (1 - discountRate)
│   (出厂价)       │
└────────┬────────┘
         │ × 1.1 (市场倍数)
         ▼
┌─────────────────┐
│    市场价格      │  marketPrice = factoryPrice × 1.1
│   (销售价)       │
└─────────────────┘
```

## 7. 评分系统详解

### 7.1 齿轮箱评分体系 (100分制)

```
┌────────────────────────────────────────────────────────────────┐
│                    齿轮箱综合评分 (100分)                        │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  1. 容量余量评分 (40分)                                         │
│     ├── 5-20% 余量    → 40分 (最佳)                            │
│     ├── 20-30% 余量   → 30分 (可接受)                          │
│     ├── 30-50% 余量   → 20分 (放宽标准)                         │
│     └── 0-5% 余量     → 25分 (小余量)                           │
│                                                                │
│  2. 减速比匹配评分 (30分)                                       │
│     ├── ≤3% 偏差      → 30分 (精确匹配)                         │
│     ├── 3-7% 偏差     → 25分 (良好匹配)                         │
│     ├── 7-12% 偏差    → 20分 (可接受)                           │
│     ├── 12-18% 偏差   → 15分 (较宽匹配)                         │
│     └── 18-25% 偏差   → 10分 (边界匹配)                         │
│                                                                │
│  3. 成本效益评分 (20分)                                         │
│     └── 基于 价格/容量 比值的标准化评分                          │
│         priceScore = 20 × (1 - (PPC - minPPC) / range)        │
│                                                                │
│  4. 推力满足评分 (10分)                                         │
│     ├── 满足推力要求   → 10分                                   │
│     ├── 无推力要求     → 5分                                    │
│     └── 不满足推力     → 0分                                    │
│                                                                │
│  5. 特殊打包价格加分 (15分) - GW系列专享                         │
│     └── 有特殊打包价格 → +15分                                  │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 7.3 排序优先级

#### 齿轮箱排序规则
1. **特殊打包价格优先** - 有特殊打包价格的型号排在前面
2. **完全匹配优先** - 完全符合条件的优先于部分匹配
3. **评分高低** - 分数差>5分时按评分排序
4. **容量余量** - 评分相近时，余量接近15%的优先
5. **价格低** - 以上条件相同时，价格低的优先

#### 联轴器排序规则
1. **评分高低** - 按综合评分降序
2. **扭矩余量** - 评分相同时，余量接近15%的优先
3. **价格低** - 以上条件相同时，价格低的优先
